libao

Someone requested a "buffer_size" know to control the latency. But this can't work, because it's global in a config file, thus common to to all programs. Either low-latency programs would get huge latency or typical mp3 players would start stuttering.
  • audio/mpg123
  • audio/vorbis-tools
  • audio/speech-dispatcher

portaudio

There are synchronisation problems with sndio, but the same seem to happen with alsa. So it's not clear if the problem is caused by the sndio bits or by audacity itself. I was told that OS X has no problems.
  • audio/audacity
  • telephony/pjsua

pulseaudio

The following are using it, but we'de better port these to sndio and avoid one extra software layer. There's no point if stacking two audio servers doing the same thing.

  • x11/gnome/controlcenter
  • x11/gnome/empathy
  • x11/gnome/settings-daemon
  • x11/gnome/shell
